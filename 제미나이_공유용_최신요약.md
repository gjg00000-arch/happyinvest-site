## 2. 배포 도메인·API 베이스 (2026-05 정리)

- **별 호스트**(스테이징·미러 도메인)로 올릴 경우 `SITE_ORIGIN` 재지정 후 `npm run build:seo` · `npm run build:og` 재실행이 필요하다.

## 4. 기술 구조(한 줄 요약)

- **정적 HTML** + `assets/common.css`, `assets/site-ux.css`, `assets/site-ux.js` 등 공통 자원.
- **하이브리드 UI**: 메인 `#cms-home-slot` 등은 API에서 HTML 조각을 주입할 수 있는 패턴이 있다.
- **결제·환불·쿠폰·가입 완료** 등은 `meta api-base`가 가리키는 API와 `fetch`로 연동한다.
- 일부 가입 플랜에서는 **약관 확인·전자서명** 후 `POST /api/legal/acceptances` 패턴으로 기록된다(`billing/index.html` 모달).

## 7. 외화 송금 UI — `billing/index.html` (2026-05 반영)

- **1단계** 납부 통화(`#wire-ccy-select`), **2단계** 수취 계좌 국가·지역(`#wire-country-select`). 통화에 따라 선택 가능한 지역만 남는다.
- **계좌 예시**: USD·KR = KEB Hana SWIFT, USD·US = Community Federal Savings Bank(CFSB), USD·SG = DBS, EUR·DE = Banking Circle IBAN, CAD/AUD/HKD/GBP 등 지역별 카드.
- **레거시** 정적 블록 `#aw-usd-base`(하나은행 단독 표)는 마크업에 남아 있으나, 동적 UI 활성 시 **`html.js-wire-route-active`에서 `display:none`** 처리되어 **이중 표기를 피한다**. 인페이지 앵커는 **`#wire-route-picker`** 로 통일.
- **통화 변경 시** `HappyUX.toast`로 수취 지역 목록 갱신을 알린다(첫 로드 시에는 토스트 없음).
- **USD·하나은행**: Beneficiary는 **인보이스와 동일한 영문 법인명**을 우선; 발신 은행 UI가 한글만 허용하면 **기관 제한·창구 확인**을 본문에 안내.
- **EUR·DE**: 미국 CFSB용과 유사한 **점선 테두리 “IBAN 입력 예시” 카드**를 동적 블록에 추가.

## 8. 환불·이벤트 구분(요지)

- **정규·비즈니스(월)**: 이유 불문 **31일·USD 일할** 등, `legal/refund.html`과 교차.
- **이벤트·쿠폰·무료체험·리커버리**: 환불불가 계열.
- **1개월 유료 이벤트**: 결제 적용 전 **별도 동의(체크박스)** 로 강한 환불불가 고지.
- **향후 아이디어**(미구현): 로그인 기반 “지금 환불 시 예상 반환액” API 연동 — 운영·백엔드 일정에 따름.

---

## 9. 크립토·Ledger(요지)

- 메이저 코인 **우선 지원**; 기타는 **견적·승인 후** 송금. 승인·정산은 **USDT 스왑 완료액** 등 운영 규칙을 따른다는 취지로 기술되어 있다.
- Ledger: **수취·보관·잔고 조회** 중심; 자동 스왑·자동 출금·시드 저장으로 **오해되지 않게** 문구 제한.
- 관리자 보드에 Ledger 자산 탭; 로컬 개발 시 `magic-indicator-api/data/ledger-portfolio-local.json` 폴백 등(세부는 API 레포).

## 12. 보안·운영(제미나이 검토 질문에 넣기 좋은 항목)

- **`admin/`** 는 정적 경로로 존재하므로, **IP 화이트리스트·역프록시 인증·API 측 권한** 등 인프라 레이어와 반드시 교차 검증한다(경로 비공개만으로는 부족).
- 결제·개인정보 관련 **로그·보관 기간**은 API·DB 정책과 `legal/privacy.html` 정합.

## 13. Google Workspace·DNS (참고)

- 과거 `magicindicatorglobal.com` 을 Workspace **별칭 도메인**으로 두는 작업과 Route 53 TXT 레코드 보강 이야기가 있었다. **현재 TXT/SPF 상태는 콘솔에서 재확인**할 것(문서만으로 실시간 검증 불가).

## 14. Gemini에게 던질 질문 예시

1. 4채널 납부 설명과 **동적 송금 표**가 합쳐졌을 때 사용자가 “어느 계좌로 보내면 되는지” 혼동하지 않는가?
2. 정규 월 **일할 환불**과 이벤트 **환불불가**가 동일 페이지에서 충분히 분리되어 보이는가?
3. 크립토 **견적 전 선송금 금지**·**스왑 완료액 기준** 문구가 과장 없이 방어적으로 읽히는가?
4. 브랜드(MagicTrading)와 법인(행복도담 인베스트) 관계가 **결제 직전**에도 인지되는가?
5. `admin/` 노출에 대한 **기술적 완화책** 제안(IP, WAF, Basic auth, 별도 서브도메인 등)을 우선순위로 정리해 달라.
