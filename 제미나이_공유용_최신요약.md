## 7. 외화 송금 UI — `billing/index.html` (2026-05-10 반영)

- **납부 통화 콤보 순서(UX)**: `USD` 다음에 **`HKD`·`CNY`**(홍콩 DBS 멀티 안내 문구) → 이어서 `KRW`, `EUR`, `GBP`, `JPY`, `AUD`, `CAD`, `CHF`, `SGD`, `NOK`, `NZD`, `SEK`.

- **USD 수취 지역 콤보 순서**: **`US`(CFSB)** → **`HK`(홍콩 DBS 멀티)** → **`SG`(싱가포르 DBS)**. (사용자 요청: 달러 다음 두 번째 = 홍콩.)
- **홍콩 SAR — DBS Bank (Hong Kong) Limited · 멀티커런시** (`rg === "HK"`): 단일 글로벌 계좌로 **HKD, USD, CNY, EUR, AUD, CAD, CHF, GBP, JPY, NZD, SGD** 입금을 안내한다.  
  - **계좌번호** `7949839366`, **Bank Code** `016`, **Branch Code** `478`, **SWIFT/BIC** `DHBKHKHH`.  
  - 수취인 표기는 스크립트 상 `GLOBAL_ACC_NAME` = `haengbokdoam invest co ltd` (Airwallex/은행 화면과 **완전 일치** 필요).  
  - **구 홍콩 Standard Chartered HKD 전용** 행(`HK_HKD_ROWS`)은 **제거**; 홍콩 경로는 위 DBS HK 멀티만 사용.
- **로컬 네트워크 우선(동적 카드)** — 통화·수취 지역이 “현지 쌍”일 때 사용자가 **국내 송금 vs 해외(SWIFT) 송금**을 헷갈리지 않도록 UI를 보강했다.  
  - **표 행 순서**: 예) **USD·US(CFSB)** — ACH routing·Fedwire routing·은행 정보를 먼저, **SWIFT/BIC는 표 하단**; **EUR·DE** — IBAN(SEPA) 우선, SWIFT 하단; **GBP·GB** — Sort code·계좌 우선, SWIFT 하단; **HK 멀티** — 계좌·Bank Code(016)·Branch·국가 후 **SWIFT 하단**(해외·국제 송금 시).  
  - **`cardHtml(..., opts)`**: 상단 **배지**(`wire-local-badge`)·짧은 **리드 문구**(`wire-local-lead`, HTML 일부 허용)·카드 강조(`wire-bank-card--local`)·필요 시 **SWIFT 각주**(`wire-swift-foot`).  
  - **카피 방향**: USD·US → ACH·Fedwire 안내(처리 일수·수수료는 출금 은행 예외 면책 포함 가능); EUR·DE → SEPA·IBAN·Instant 가능성 언급; GBP·GB → Faster Payments 등; AUD·AU → BSB; CAD·CA → 현지·E-transfer; KRW·KR → 국내 이체(해외송금 메뉴 아님); HK → **HKD는 FPS·은행 코드 016** 등 현지 우선, **홍콩 외 거주자**에게는 SWIFT 중개·수수료 가능성·**거주지 로컬 수취 계좌** 검토 권유.  
  - **SSOT 주의**: “수수료 0원” 등은 은행·상품·경로에 따라 달라질 수 있어 문구는 **과장 없이** 방어적으로 쓴다.
- **`HappyUX.toast` (`assets/site-ux.js`)**  
  - **통화만 바뀔 때**: 기존과 같이 수취 지역 콤보 갱신 안내(첫 로드 제외).  
  - **통화·지역 조합(로컬 경로)이 바뀔 때**: `wireLocalToastMessage(ccy, rg)` — 예) USD·US는 “현지 송금 네트워크…” 류, HK 멀티는 FPS·016·해외 SWIFT 비용 인지 등 **경로별 문구**(첫 렌더는 토스트 없음, 이후 조합 변경 시만).
- **기타 통화·지역 매핑(요지)**: `EUR` = DE(IBAN) / HK / SG; `GBP` = GB / HK / SG; `AUD`·`CAD` = 각 현지 / HK / SG; `CNY`·`JPY`·`NZD`·`SGD`·`CHF` = HK / SG; `NOK`·`SEK` = SG만.
- **계좌 예시(비 HK)**: USD·US = CFSB; USD·SG·기타 SG = 싱가포르 DBS `dbsRows()`; EUR·DE = Banking Circle IBAN; CAD = Digital Commerce Bank; AUD = ANZ; GBP·GB = Airwallex UK 등 — 기존 로직 유지.

- **EUR·DE**: **점선 테두리 “IBAN 입력 예시” 카드** 동적 블록 유지.

## 7-1. 접속 폭주 시 초기 대기 안내 — 정적 트래픽 게이트 (2026-05-10)

- **목적**: 무료·이벤트 등으로 **짧은 순간 트래픽이 몰릴 때** 사용자에게 **대기·재시도 안내**를 보여 **과도한 새로고침·혼란**을 줄이는 **초기 단계** 장치. **국세청급 전역 큐·공정 순번 시스템이 아님**(Redis·백엔드 대기열 없음).
- **동작**: `assets/traffic-gate.json`의 **`enabled`** 가 `true`일 때만, 연동된 페이지에서 전면 오버레이 표시. 기본값은 **`enabled: false`** 로 일반 이용에 영향 없음.
- **연동 페이지(현재)**: `index.html`, `registration/index.html`, `billing/index.html` — 각각 `assets/traffic-wait-gate.css` + `assets/traffic-wait-gate.js`(defer, `data-gate-config`·`data-home-href`) 로드.
- **표시 내용(설정값 기반)**: `approxQueueAhead`(앞줄 추정 인원), 이전 폴링 대비 **감소 인원** 문구(세션 저장소에 이전 값 비교), `etaMinutesMin`~`etaMinutesMax`(예상 대기 **분 범위**, 참고용 면책 문구 포함), **상태 새로고침**·**메인으로** 버튼. **“내 접수 표시”** 코드는 브라우저 세션용 짧은 식별자 수준(전역 고유 순번 아님).
- **폴링**: `pollSeconds`(기본 5초 근처). `fetch(..., { cache: "no-store" })` + 쿼리 버스트로 **배포 직후 JSON 갱신**을 노림.
- **운영 시 주의**: **CloudFront 등 CDN** 사용 시 `traffic-gate.json` 경로에 **캐시 무효화**가 없으면 `enabled` 변경이 사용자에게 **늦게 반영**될 수 있음. 긴급 ON/OFF 후 **무효화 또는 짧은 TTL 정책**을 검토할 것.
- **한계·검토 포인트**: 원본 서버·API가 이미 **응답 불가**면 JSON 조회도 실패할 수 있어 **완전한 장애 대비**는 아님. **과장된 정확도**(“정확한 N분 후 입장”)로 읽히면 분쟁 리스크 — 문구는 **추정·참고**로 유지하는 것이 안전. 장기적으로는 **WAF·대기실 SaaS·API 분리** 등과 병행 검토.
