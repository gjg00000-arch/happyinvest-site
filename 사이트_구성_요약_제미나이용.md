# Magic 지표 — 사이트·운영 통합 요약 (제미나이 공유용)

**문서 목적**: `magic-indicator-site` 정적 홈페이지와 `magic-indicator-api` 운영 정책의 현재 상태를 제미나이 등 외부 검토용으로 빠르게 전달하기 위한 요약이다. 법무 원문·금액·API 동작의 최종 기준은 각 HTML, API README, 서버 코드 원본을 우선한다.

## 2. 가격·플랜 표준

| 구분 | 이벤트 1개월 | 정규 월 |
|------|--------------|---------|
| Whop TRV | `$119` | `$5,499` |
| Whop MT5 | `$129` | `$5,999` |
| MQL MT5 | `$149` | `$6,999` |

- 정규는 연·분기·다월 선납·년회원 상품을 두지 않고 월 1회 납부를 원칙으로 한다.
- 쿠폰으로 이용 기간이 늘어나는 것은 결제 주기와 별개다.
- 7일 무료 체험은 자동 유료 청구되지 않는다.
- 1개월 유료 이벤트는 안내·결제·활성화 이후 환불불가로 별도 고지한다.

## 3. 결제 채널 정리

홈페이지 공개 문구는 `세 갈래`가 아니라 4채널로 정리했다.

| 채널 | 용도 |
|------|------|
| PayPal | 소액, 이벤트, 가입 단계 카드 결제 중심. PayPal 정책에 따라 잔액·페이류 제한 가능 |
| Airwallex | 고액, 정규, 다통화 직접 송금, 인보이스 중심 |
| 크립토 | 온체인 직접 송금. 메이저 우선 지원, 기타 코인은 수동 견적 |
| 국내 원화 법인 송금 | 국내 원화 납부. 당사 법인 계좌와 인보이스 기준 |

문구 정합성 정리:

- `메이저 자동`은 과하게 보일 수 있어 `메이저 우선 지원`으로 정리.
- `모든 코인 지원`은 실제 거절 가능성과 충돌하므로 `메이저 우선·기타 코인 견적 정책`으로 정리.
- 공개 페이지의 주소 표는 실제 송금 기준이 아니라 `인보이스 안내` 기준으로 표시.
- XRP의 `Tag 필수` 표현은 체인·거래소 조건에 따라 달라질 수 있어 `Tag/메모는 인보이스 기준`으로 정리.
- `PayPal+Airwallex`처럼 결제 흐름이 섞여 보이는 표현은 `당시 켜진 PG와 운영 수동 대조`로 정리.

## 4. 크립토·Ledger 정책

메이저 우선 지원 코인:

- BTC, ETH, USDT, USDC, XRP, SOL, TRX, BNB, MATIC
- 내부 Ledger 자산관리에는 Base, Arbitrum, Optimism, Avalanche C-Chain, Dogecoin, Litecoin 주소도 등록됨

기타·잡코인 처리:

- Ledger 지원 코인, 신규 코인, 밈코인, 저유동성 토큰은 먼저 보내면 안 된다.
- 먼저 코인명, 네트워크, 예정 수량으로 견적 요청을 받는다.
- 운영이 유동성, 매도 가능 여부, 스왑 수수료, 네트워크 수수료를 확인한다.
- 견적 유효시간과 수취 방식을 안내한 뒤 송금한다.
- 실제 결제 승인은 입금 시점 표시가가 아니라 실제 `USDT 스왑 완료액` 기준이다.
- 허니팟, 매도 불가, 출금·스왑 곤란, 극저유동성 토큰은 접수 후에도 거절 가능하다.
- Ledger는 자동 스왑 장치가 아니다. 입금 감지·운영 알림은 자동화할 수 있지만 출금·스왑은 운영자가 확인하고 직접 승인한다.

Ledger 자산관리 현황:

- 관리자 보드에 `Ledger 자산` 탭이 있다.
- 로컬 개발에서는 MongoDB 없이 `magic-indicator-api/data/ledger-portfolio-local.json`으로 폴백 저장한다.
- 등록된 주소/xpub과 수동 수량·USDT 단가를 기반으로 포트폴리오를 보여준다.
- 지원 체인은 서버가 잔고·가격·환율 조회를 시도하고, 실패·미지원 자산은 수동값으로 평가한다.
- stablecoin인 USDT/USDC는 단가가 비어 있으면 `1 USDT` 기준으로 보정한다.
- TRC-20 USDT, EVM 토큰, BTC, XRP, SOL, TRX, DOGE, LTC 등 조회 로직이 추가되어 있다.

## 5. 환불·법무 요지

| 항목 | 기준 |
|------|------|
| 정규·비즈니스 월 | 이유를 묻지 않는 31일·USD 일할 환불 |
| 사용일 | 구간 시작일부터 환불 신청 당일까지, 신청 당일 포함 |
| 해외 수취 | USD 지급 원칙 |
| 국내 수취 | 동일 USD 환불액을 기준으로 당시 운영 안내에 따라 원화 입금 가능 |
| 이벤트·쿠폰·체험·리커버리 | 환불불가 |
| 1개월 유료 이벤트 | 환불·취소·대가 반환 불가로 별도 강하게 고지 |

주의:

- `환불불가(원칙)`과 `절대 환불 불가`가 같이 쓰인다. 전자는 이벤트성 혜택 일반 원칙, 후자는 1개월 유료 이벤트의 별도 강한 예외 고지다.
- 정규 환불과 이벤트 환불불가를 혼동하지 않도록 각 페이지에서 `정규·비즈니스 월만`이라고 반복 고지한다.

## 8. 최근 문장 패러독스 감사 결과

| 기존 문제 | 수정 방향 |
|-----------|-----------|
| 홈에서 `세 갈래(초안)`이라고 하면서 실제로는 4채널을 설명 | `납부·결제는 네 갈래`로 수정 |
| `준비중 · 채택 전 초안`처럼 현재 운영과 맞지 않는 홈 문구 | `운영 안내 · 결제 채널`로 수정 |
| `모든 코인 지원`과 `거절 가능`이 동시에 존재 | `메이저 우선·기타 코인 견적 정책`으로 수정 |
| `메이저 자동` 표현이 실제 수동 대조와 충돌 | `메이저 우선 지원`으로 수정 |
| Ledger 관리자 화면이 `도착 전 준비 화면`이라고 표시 | 현재 상태에 맞게 `Ledger 자산현황`으로 수정 |
| 법무 요약의 `묻지 않는(묻지 않는)` 중복 | `이유를 묻지 않는 일할 환불`로 정리 |
| XRP `Tag 필수`가 모든 상황에 고정된 것처럼 보임 | `Tag/메모는 인보이스 기준`으로 수정 |
| `PayPal+Airwallex` 정산 표현이 결제 흐름을 혼동시킴 | `당시 켜진 PG와 운영 수동 대조`로 수정 |

남겨둔 표현:

- `준비중`, `미연동`은 실제 PG ON/OFF 상태 표시나 다운로드 URL 미연결 안내에 해당하므로 유지 가능.
- `환불불가(원칙)`과 `절대 환불 불가`는 법무 고지 강도 차이로 유지 가능.

## 9. 제미나이에 요청할 때 유용한 질문

1. 이 홈페이지의 결제·환불·크립토 정책 문구 사이에 사용자 오해를 만들 수 있는 모순이 남아 있는가?
2. `메이저 우선 지원·기타 코인 수동 견적` 정책이 법무·결제·환불 페이지에서 일관되게 보이는가?
3. `정규 월 환불 가능`과 `이벤트 환불불가`의 구분이 충분히 명확한가?
4. Ledger 관련 문구가 사용자가 자동 스왑이나 자동 출금을 기대하지 않도록 충분히 제한적인가?
5. 공개 페이지에서 실제 수취 주소를 숨기고 인보이스 기준으로 안내하는 방식이 결제 오인 가능성을 줄이는가?
